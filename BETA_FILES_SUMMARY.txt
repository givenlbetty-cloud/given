🔐 SÉCURITÉ - RÉSUMÉ
═══════════════════════════════════════════════════════════════════════════

✅ Protection 1: BASIC AUTH
   Chaque accès demande un mot de passe
   → Activé si ENABLE_BASIC_AUTH=True

✅ Protection 2: CONFIGURATION SÉPARÉE
   settings_beta.py complètement indépendant
   → Production ne partage aucune config

✅ Protection 3: BD ISOLÉE
   db_beta.sqlite3 ≠ db.sqlite3
   → Zéro risque pour production

✅ Protection 4: MÉDIAS SÉPARÉS
   media_beta/ ≠ media/
   → Uploads de test ne touchent pas prod

✅ Protection 5: SERVICES TEST
   Stripe TEST keys (sk_test_) seulement
   Email via console (pas d'envoi réel)
   → Zéro coût, zéro impact

✅ Protection 6: DOMAINE ISOLÉ
   ALLOWED_HOSTS=beta.monprojet.com uniquement
   → Autres domaines rejetés

🚀 ÉTAPES DE DÉMARRAGE
═══════════════════════════════════════════════════════════════════════════

1️⃣  PRÉPARER (1 minute)
   cp .env.beta.example .env.beta
   nano .env.beta
   # Changer SECRET_KEY, BASIC_AUTH_PASSWORD, vérifier STRIPE

2️⃣  VALIDER (1 minute)
   python3 validate_security.py
   # Doit afficher: ✅ ENVIRONNEMENT BETA SÉCURISÉ

3️⃣  DÉPLOYER (2 minutes)
   bash deploy_beta.sh
   # Crée db_beta.sqlite3, media_beta/, staticfiles_beta/

4️⃣  LANCER (instant)
   bash run_beta.sh
   # Serveur sur http://localhost:8000

5️⃣  ACCÉDER (instant)
   Navigateur: http://localhost:8000
   Basic Auth: beta_user / [votre password]
   Admin: http://localhost:8000/admin/
   Admin login: admin_beta / beta_admin_password_123
